#  Investigación sobre Markdown y su uso en proyectos de software

---

##  1. ¿Qué es Markdown y por qué se utiliza en proyectos de software?

**Markdown** es un lenguaje de marcado ligero que permite escribir documentos en texto plano con formato sencillo.  
Fue creado para ser fácil de leer incluso sin procesarlo.

En proyectos de software se utiliza porque:

- Facilita la creación de documentación técnica.
- Funciona en cualquier editor de texto.
- Es rápido de escribir y mantener.
- Se integra perfectamente con Git y GitHub.
- Permite crear archivos como README.md, documentación, manuales y wikis.

---

##  2. Ejemplo práctico de uso de Markdown

###  Encabezados

```md
# Título nivel 1
## Título nivel 2
### Título nivel 3
```

###  Listas

**Lista ordenada:**

```md
1. Primer paso
2. Segundo paso
3. Tercer paso
```

**Lista no ordenada:**

```md
- Elemento A
- Elemento B
- Elemento C
```

##  3. Ventajas de utilizar Markdown en combinación con GitHub

- GitHub renderiza Markdown automáticamente.  
- La documentación se ve profesional en el repositorio.  
- Facilita la colaboración y el control de versiones.  
- Permite crear páginas web con GitHub Pages.  
- Es ideal para issues, wikis y pull requests.  
- Es ligero, portable y fácil de editar.  

---

##  Conclusión

Markdown es una herramienta simple pero poderosa para la documentación en proyectos de software.  
Su sintaxis ligera permite a desarrolladores y equipos crear y mantener documentación legible y versionable sin barreras técnicas.  
Combinado con GitHub, Markdown facilita la colaboración, la revisión y la publicación de contenido técnico, mejorando la comunicación y la eficiencia en el ciclo de vida del desarrollo.  
Adoptar Markdown como estándar de documentación aporta claridad, consistencia y portabilidad a cualquier proyecto, desde pequeños repositorios hasta grandes codebases con múltiples colaboradores.
